# Phase 2.5: Multi-Stream Color, Calibrated FoV, and 30 fps Overlap — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Checkbox steps.

**Goal:** Clear the top three deferred items from Phase 2: (1) ingest the newly added VL53L9CX FoV/thermals datasheets and calibrate the Deprojector's placeholder FoV; (2) expose the transform's reflectance/confidence/ambient/ZAPC outputs host-side with a viewer `--color` mode; (3) restructure the firmware acquisition loop so sensor integration overlaps the CDC send (~24.6 → ~30 fps).

**Architecture:** All host-side work builds on the existing shim/wrapper/pipeline; the shim grows an output-selection mask API (breaking the internal DLL API is fine — single consumer). ZAPC (on-device-grade deprojection, calibrated intrinsics, computed on the PC from the same raw) becomes the ground truth to validate/correct the Deprojector's linear-FoV model. Firmware change is confined to the raw-only loop in `vl53l9_app.c`; the dual-stream golden-pair mode is NOT touched.

## Global Constraints

- All Phase 1/2 conventions stand: 53L9A1 read-only; protocol v1 additive only (no wire change is expected in this plan — everything new is host-side or firmware-internal); commits `git -c commit.gpgsign=false` + the session trailers; ARM toolchain + flash paths per `.superpowers/sdd/` reports; board on COM14/CDC CAFE:4001; suite baseline 39 passing.
- Full 54×42 resolution remains a hard requirement.
- Output formats for reflectance/confidence/ambient are whatever the transform library negotiates — **read `docs/transform-streams.md` for the capability formats; do not assume float32.** ZAPC = 4×float32 [x,y,z,confidence]/zone (verified in Phase 2 docs).
- Equivalence protection: the depth output path (ZF32) must remain byte-identical through the shim changes — the equivalence test (`test_equivalence.py`) is the regression net and must stay green.

### Task 2: Multi-output transform + viewer --color

**Files:** `host/transform/rs_transform_shim.{c,h}`, `host/src/roomscan/native.py`, `pipeline.py`, `viewer.py`; tests `host/tests/test_native_multi.py`, extend `test_pipeline.py`.

**Interfaces:**
- Shim v2: `rst_create2(calib, calib_len, in_w, in_h, uint32_t out_mask) -> void*` and `rst_process2(h, raw, raw_len, float *depth, void *reflectance, void *confidence, void *ambient, float *zapc) -> int` — mask bits: DEPTH=1, REFLECTANCE=2, CONFIDENCE=4, AMBIENT=8, ZAPC=16; NULL pointers for unselected outputs; aux buffer dtypes per the negotiated formats (document each in the header). Keep `rst_create/rst_process/rst_destroy` as depth-only wrappers over v2 (equivalence test untouched).
- ZAPC note: it's a FORMAT of the depth stream, not a separate stream (Phase 2 finding) — the shim likely needs a second transform instance OR re-negotiation to emit both ZF32 and ZAPC; investigate the capability model (`docs/transform-streams.md`, `vl53l9_transform.c` read-only) and pick the cheapest correct approach; if two instances are needed, document the memory/time cost. DEPTH|ZAPC together is only needed by Task 3's validation — the live viewer never needs both.
- Python: `Transform(calib, outputs=("depth",))` — tuple of names; `.process(raw)` returns a dict of ndarrays keyed by output name (backward-compat: existing single-output call sites updated in the same commit).
- `TransformStage(outputs=...)`: passes through; result payload becomes `(header, {name: array})`; viewer `--color {depth,reflectance,confidence}` selects the colorize channel (depth = existing z-based turbo; reflectance/confidence = turbo over the aux plane normalized per-frame); default `depth` (no behavior change).
- Tests (DLL-gated where needed): mask selection returns exactly the requested planes with correct shapes/dtypes; determinism across fresh instances on golden raw; depth plane from multi-output == depth from legacy single-output (byte-identical); pipeline carries aux planes; viewer arg plumbed (smoke: supervised replay of the bundled recording with `--color reflectance` — window + HUD + no traceback).
